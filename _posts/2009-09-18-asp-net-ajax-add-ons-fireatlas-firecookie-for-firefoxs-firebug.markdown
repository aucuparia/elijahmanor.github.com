<p>Today’s screencast is the 4th in a series of developer Firefox Extensions. You can view my previous 3 short <a href="http://screenr.com/user/elijahmanor" target="_blank">Screenr</a> screencasts here… </p>  <ul><li><a href="http://elijahmanor.com/webdevdotnet/post/Firefinder-Add-on-for-Firebug-Add-on-for-Firefox.aspx" target="_blank">Firefinder for Firebug for Firefox for Fiddling with jQuery</a> </li>    <li><a href="http://elijahmanor.com/webdevdotnet/post/Digging-Into-The-FireQuery-Add-on-for-jQuery.aspx" target="_blank">Digging Into The FireQuery Add-on for jQuery</a> </li>    <li><a href="http://elijahmanor.com/webdevdotnet/post/Google-Page-Speed-Yahoo-YSlow-Firebug-Extensions-to-Increase-Web-Page-Performance.aspx" target="_blank">Page Speed & YSlow Firebug Extensions to Increase Web Page Performance</a> </li> </ul><p>For those of you that do ASP.NET or ASP.NET AJAX development, then the following <a href="https://addons.mozilla.org/en-US/firefox/addon/1843" target="_blank">Firebug</a> Add-ons might be of interest to you. </p>  <p><a href="https://addons.mozilla.org/en-US/firefox/addon/12121" target="_blank">FireAtlas</a> is a tool that enables you to view the PageRequestManager events, trace WebService calls, inspect Partial Updates, and view any installed application components. You can actually try out some <a href="http://fireatlas.chesnaistechnologies.com/" target="_blank">samples pages</a> to exercise most of the functionality of this Add-on, but I will be viewing the twitter scheduler application <a href="http://www.twtmstr.com/" target="_blank">TwtMstr</a> (developed by <a href="http://twitter.com/wbm" target="_blank">@wbm</a>) to demonstrate some of the features in my screencast.</p>  <p><a href="https://addons.mozilla.org/en-US/firefox/addon/6683" target="_blank">FireCookie</a> is another useful tool that allows you to view any cookies that are relevant for the current web page. This can be handy for an ASP.NET developer to inspect the <a href="http://msdn.microsoft.com/en-us/library/system.web.configuration.rolemanagersection.cookiename(VS.100).aspx" target="_blank">.ASPXROLES</a>, <a href="http://msdn.microsoft.com/en-us/library/system.web.configuration.formsauthenticationconfiguration.name(VS.100).aspx" target="_blank">.ASPXAUTH</a>, or <a href="http://msdn.microsoft.com/en-us/library/system.web.configuration.sessionstatesection.cookiename.aspx" target="_blank">ASP.NET_SessionId</a> values that are common for most ASP.NET applications.</p>   <p> </p>  <p>Yet again, I hope this was beneficial to you.
