1. What is this?
  • Whether you are a beginner or intermediate jQuery developer, if you don't have a good understanding of the this statement then you'll most likely find yourself stumbling along.
1. What is this?
  • Under normal circumstances this refers to a DOM element
/* Binding to an event */
$( "a.demo" ).click( function( e ) {
    alert( this.href );

    e.preventDefault();
});
  • Google
  • Microsoft
1b. What is this?
  • Exception #1: jQuery Plugin 
(function( $ ) {
    $.fn.myPlugin = function() {
        /* this refers to jQuery object from jQuery selection */
        alert( "this.length = " + this.length );
        return this.each( function() {
            /* this refers to DOM element from jQuery selection */
            alert( "this.id = " + this.id );
        });
    }
})( jQuery );


$( "a.demo" ).myPlugin();
  • Google Microsoft
1c. What is this?
  • Exception #2: $.proxy()new in jQuery 1.4 
var twitter = {
    userName: "elijahmanor",
    tweet: function() { alert( "Hello " + this.userName ); } 
};

/* Using event helper, bind(), or live() */
$( "#update1" ).click( twitter.tweet ); /* Undefined */

/* Option #1: jQuery.proxy( function, context ) */
$( "#update2" ).click( $.proxy( twitter.tweet, twitter ) );

/* Option #2: jQuery.proxy( context, name ) */
$( "#update3" ).click( $.proxy( twitter, "tweet" ) );
2. $ Object | DOM Element
  • Now that we know what this refers to, it doesn't end the confusion
  • It is common practice to wrap this in a jQuery object and save it in a variable for later use
var jQueryWrappedElement = $( this );
  • The confusion comes if a developer forgets if the variable refers to the jQuery Object or the DOM Element
2a. $ Object | DOM Element
  • To alleviate this confusion some prepend a '$' to the variable where they save the value of this.
    • Hungarian Notation - Good, Bad, Ugly...
$( "a" ).click( function() {
    var $anchor = $( this );

    $anchor.fadeOut();
})
  • Note: This is highly debated, but if it helps you... go ahead & use it ;)
2b. $ Object | DOM Element
  • Wrapping the DOM element with the jQuery object
$( "a" ).click( function( e ) {
    var anchor = $( this );

    anchor.fadeOut();

    e.preventDefault();
})
2c. $ Object | DOM Element
  • Unwrapping the DOM element from the jQuery object
//Gets collection of anchor tags from jQuery array
$( "a" ).get();

//Get method returning jQuery array & indexing 0th item
$( "a" ).get()[ 0 ];

//Get method accessing DOM element at 0th index
$( "a" ).get( 0 );

//Array accessor syntax getting the 0th indexed item
$( "a" )[ 0 ];
3. Knowing Selectors
  • Selectors are really the beginning place for most of what you do in jQuery
    • You find something & then you perform some action on it
  • In order to be a proficient jQuery developer, you must know the full extent of jQuery's powerful selectors
3a. Knowing Selectors
  • Simple selectors
//Selects all the anchor tags in the DOM
$( "a" )

//Selects all the anchor tags inside of div elements in the DOM
$( "div a" )

//Selects all the tr elements with a class of 'odd'
$( "tr.odd" )
3b. Knowing Selectors
  • Uncommon selectors
//Selects all links that ends with mp3 in the href
$( "a[href$='mp3']" ) 

//Selects all links that are set to open a new window
$( "a[target='_blank']" ) 

//Selects all links that have a class that begins with "custom"
$( "a[class^='custom']" )

4c. Dos & Don'ts of Events
  • Event Helpers
    • click, dblclick, mouseover, etc...

4d. Dos & Don'ts of Events
  • bind() & unbind()
var clickMe = function() {
    console.log( "Hello World!" );
};

$( "p.clickMe" ).bind( "click dblclick", clickMe );

// really dangerous
$( "p.clickMe" ).unbind();

// moderately dangerous
$( "p.clickMe" ).unbind( "click dblclick" );          

// safe
$( "p.clickMe" ).unbind( "click dblclick", clickMe );

4h. Dos & Don'ts of Events
  • jQuery 1.7 rewrote events from the group up and introduced two new binding methods called on and off
    • The bind/live/delegate methods are now just one line pass throughs to the new on & the same applies to unbind/die/undelegate mapping to the off method.
// behave like bind
$( ".customer" ).on( "click", function( e ) {} );                

// behave like live
$( document ).on( "click", ".customer", function( e ) {} );      

// behave like delegate
$( ".customers" ).on( "click", ".customer", function( e ) {} );
5. Ways of Storing Data
  • It is common practice to store additional information about your DOM elements
  • Fortunately, there are various ways of doing this
    • Some are better than others...
5a. Ways of Storing Data
  • Don't Use Attributes
// Storing special info about the anchor in the alt attribute
$( "a" ).attr( "alt", "Special info about this HTML tag" );

// Getting the special information from the anchor's alt attribute
$( "a" ).attr( "alt" )
5b. Ways of Storing Data
  • Utilizing the jQuery data() Method - Storing data into jQuery's internal data store
    • Simple Data Types
$( "li.demo" )
    .data( "specialInfo", "Special info about this HTML tag" );

alert( $( "li.demo" ).data( "specialInfo" ) );
5c. Ways of Storing Data
  • Utilizing the jQuery data() Method
    • Complex Data Types
/* Storing a complex data structure under the specialInfo key */
$( "li.demo" )
    .data( "specialInfo", {
        firstName: "Elijah",
        lastName: "Manor"
    });

/* Getting the complex specialInfo & accessing the firstName */
alert( $( "li.demo" ).data( "specialInfo" ).firstName );
  • Note: Check out FireQuery Plug-in for Firebug
5d. Ways of Storing Data
  • Utilizing the jQuery data() Method
    • Alternate syntax...
$( "li.demo" ).data( { firstName: "Elijah", lastName: "Manor" } );

console.dir( $( "li.demo" ).data() );

$( "li.demo" ).data( "someKey",
    { firstName: "Andrea", lastName: "Manor" } );

console.dir( $( "li.demo" ).data() );

console.log( $( "li.demo" ).data().someKey.firstName );
  • Note: Check out FireQuery Plug-in for Firebug
5e. Ways of Storing Data
  • Utilizing the jQuery data() Method
    • Removing the data()
$( "li.demo" )
    .data( "specialInfo", "Special info about this HTML tag" );

alert( "Before removeData(): " + $( "li.demo" ).data( "specialInfo" ) );

$( "li.demo" ).removeData( "specialInfo" );

alert( "After removeData(): " + $( "li.demo" ).data( "specialInfo" ) );
5f. Ways of Storing Data
  • HTML5 data- Attributes
<div data-role="page" 
    data-last-value="43" 
    data-hidden="true" 
    data-options='{"name":"John"}'></div>

Support added in 1.4.3 and in 1.6 change was made to conform to W3C HTML5 specification ( embedded dashes )

$( "div" ).data( "role" ) === "page";
$( "div" ).data( "lastValue" ) === 43;
$( "div" ).data( "hidden" ) === true;
$( "div" ).data( "options" ).name === "John";

jQuery attemps to convert the string to the desired value. If you want the string, then you can use the attr() method.

There is a great article about all the ins & outs of this topic by Dave Methvin entitled Using jQuery’s Data APIs.

  • Low Level AJAX Method

$.ajax({
    type: 'POST',
    url: "/Pet/Create",
    data: formData,
    success: function ( data, textStatus, xhr ) {} );
    // etc...
});

There are a lot of properties you can set in the $.ajax method...

  • async, beforeSend, cache, complete, contentType, context, data, dataFilter, dataType, error, password, success, timeout, url, username, etc...
6d. Ins & Outs of AJAX
  • Deferreds

jQuery 1.5 added the concept of Deferreds. Instead of having to add your callbacks to success and error during the definition of your AJAX call, you can now decide to define those later or append to them.

// Save off the special jQuery XMLHTTPRequest Deferred 
var jqXhr = $.ajax( "/Pet/Details/3" ) 
    .done( function() { alert( "success" ); })
    .fail( function() { alert( "error"); })
    .always( function() { alert( "complete" ); });

jqXhr.always( function() { alert( "second complete" ); });
6e. Ins & Outs of AJAX
  • When Method

Sometimes you want to know when several async operations have finished before doing something else. The $.when method is useful for such scenarios.

$.when( $.ajax( "/page1.php" ), $.ajax( "/page2.php" ) )
    .then( successCallback, failureCallback );
